# Rawlings Tigers NOVA Hugo Website

This package is a Hugo-based redesign starter for the Rawlings Tigers NOVA website. It is built for static hosting on Cloudflare Pages or GitHub Pages, with a unified Rawlings Tigers / NOVA visual system and data-driven pages for the content that changes most often.

## Design direction

The active theme uses one shared design system instead of page-specific embedded HTML. The design is inspired by college/pro baseball program sites but themed for Rawlings Tigers NOVA:

- black/dark hero sections
- Rawlings Tigers orange accent color
- wide responsive desktop container
- mobile-first card layouts
- reusable team, coach, sponsor, tryout, and accolade cards
- consistent header, footer, hero, section heading, CTA, and grid patterns

## Prerequisites

Install Hugo Extended.

### macOS with Homebrew

```bash
brew install hugo
hugo version
```

### Windows with Chocolatey

```powershell
choco install hugo-extended
hugo version
```

## Run locally

From the project root:

```bash
hugo server -D
```

Then open:

```text
http://localhost:1313/
```

If you want to reset generated output before testing:

```bash
rm -rf public resources .hugo_build.lock
hugo server -D
```

## Validate local assets

Before deploying, run:

```bash
./scripts/validate-local-assets.sh
```

You can also manually check that no TeamLinkt CDN media references remain:

```bash
grep -R "cdn-app\.teamlinkt\.com" -n . --exclude-dir=.git --exclude-dir=public
```

That command should return no source file matches.

## Build the static site

```bash
hugo --minify
```

The generated site will be in:

```text
public/
```

Do not edit files inside `public/` directly. They are generated by Hugo.

## Deploy with Cloudflare Pages

1. Create a GitHub repository for this site.
2. Commit and push this project.
3. In Cloudflare, go to **Workers & Pages**.
4. Select **Create application** → **Pages** → **Connect to Git**.
5. Choose the GitHub repository.
6. Use these build settings:

| Setting | Value |
|---|---|
| Framework preset | Hugo |
| Build command | `hugo --minify` |
| Build output directory | `public` |
| Root directory | `/` |

Recommended environment variable:

| Variable | Example value |
|---|---|
| `HUGO_VERSION` | `0.163.0` |

After deploy, add the custom domain in Cloudflare Pages and follow the DNS instructions Cloudflare provides.

## Deploy with GitHub Pages

Cloudflare Pages is recommended, but GitHub Pages also works.

1. Push the repository to GitHub.
2. In GitHub, go to **Settings** → **Pages**.
3. Set source to **GitHub Actions**.
4. Add a Hugo workflow under `.github/workflows/hugo.yaml`.

Example workflow:

```yaml
name: Deploy Hugo site to Pages

on:
  push:
    branches: ["main"]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: "pages"
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: '0.163.0'
          extended: true
      - name: Build
        run: hugo --minify
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./public
  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    runs-on: ubuntu-latest
    needs: build
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

## How to update the site

### Update teams

Edit:

```text
data/teams.yaml
```

Each team entry controls the Team Central cards: team name, status, coach, coach photo, record, and links.

### Update coaches

Edit:

```text
data/coaches.yaml
```

Head coach bios and assistant coach listings are managed here.

### Update tryouts

Edit:

```text
data/tryouts.yaml
```

Tryout schedule, evaluation areas, age chart image, registration URL, and FAQ content are managed here.

### Update sponsors

Edit:

```text
data/sponsors.yaml
```

Sponsorship packages, contact details, and impact bullets are managed here.

### Update accolades

Edit:

```text
data/accolades.yaml
```

Team achievements, banners, and player highlight images are managed here.

### Update normal content pages

Edit files under:

```text
content/
```

Examples:

```text
content/about.md
content/family-hub.md
content/booster-club.md
content/watch-now.md
```

### Update navigation

Edit:

```text
data/navigation.yaml
```

### Update shared styling

Edit:

```text
assets/css/main.css
```

The site is intentionally built around shared reusable classes so the pages stay visually consistent.

## v7 Design Direction: Twelve Baseball-Inspired NOVA Theme

This version moves the visual system closer to the premium travel-ball style used by high-level baseball organizations such as Twelve Baseball, while keeping Rawlings Tigers NOVA branding.

Key layout changes:

- White primary navigation with a dark utility/social bar.
- Large editorial home hero with oversized athletic typography.
- Dark navy/black and Rawlings Tigers orange palette.
- Strong program statement band below the hero.
- Wider responsive layout for desktop, but card-first mobile behavior.
- More consistent page rhythm: hero, intro, card/grid content, CTA/footer.
- Team Central remains data-driven and card-based instead of table-first.

The design is not a purchased or copied template. It is a custom Hugo theme direction inspired by premium travel baseball and college/pro baseball site patterns.

## v8 homepage/header/footer refresh

This package moves the site closer to the Twelve Baseball-inspired direction while keeping Rawlings Tigers NOVA branding:

- `layouts/partials/site-header.html` is the global header used by every page.

## Local Icon System

The site uses local SVG icons stored in:

```text
assets/icons/bootstrap/
```

Icons are rendered through:

```text
layouts/partials/icon.html
```

Use icon names in YAML, for example:

```yaml
icon: trophy
icon: graph-up-arrow
icon: clipboard-check
icon: people
icon: house-heart
icon: award
```

The header and footer social icons are also local SVG assets, so Cloudflare Pages does not depend on Font Awesome, Bootstrap Icons CDN, or any third-party icon service at runtime.

## Icon attribution

This site uses a small locally bundled set of Game-icons.net icons for baseball/program UI graphics. They are stored in `assets/icons/game-icons/` and credited on `/credits/`. Keep the footer `Icon Credits` link unless you replace the icon set with a license that does not require attribution.
